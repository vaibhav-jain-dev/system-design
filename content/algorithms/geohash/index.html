"Geohash: O(log N) B-tree index lookup across 9 cells. Haversine scan: O(N) full table scan. At 1M restaurants, log N ≈ 20 vs 1,000,000 operations. The geohash index is ~50,000x faster. Always query self + 8 neighbors." }} {{phase 2 "Encoding Algorithm" "5 min"}} {{say "Geohash encodes by repeatedly bisecting the coordinate space. Latitude range [-90, 90] and longitude range [-180, 180] are halved alternately, building a binary string that is then base32 encoded."}} {{code "python" `BASE32 = "0123456789bcdefghjkmnpqrstuvwxyz" def encode(lat: float, lon: float, precision: int = 6) -> str: """ Encode (lat, lon) as a geohash string of given precision. Each base32 char encodes 5 bits. precision=6 → 30 bits → ~1.2 km × 0.6 km cell. """ lat_range = [-90.0, 90.0] lon_range = [-180.0, 180.0] bits = [] is_lon = True # start with longitude (alternating: lon, lat, lon, lat...) for _ in range(precision * 5): if is_lon: mid = (lon_range[0] + lon_range[1]) / 2 if lon >= mid: bits.append(1) lon_range[0] = mid else: bits.append(0) lon_range[1] = mid else: mid = (lat_range[0] + lat_range[1]) / 2 if lat >= mid: bits.append(1) lat_range[0] = mid else: bits.append(0) lat_range[1] = mid is_lon = not is_lon # Pack bits into base32 result = "" for i in range(0, len(bits), 5): index = int("".join(str(b) for b in bits[i:i+5]), 2) result += BASE32[index] return result def decode(geohash: str) -> tuple[float, float]: """Return (lat, lon) center of geohash cell.""" lat_range = [-90.0, 90.0] lon_range = [-180.0, 180.0] is_lon = True for char in geohash: index = BASE32.index(char) for bit_pos in range(4, -1, -1): bit = (index >> bit_pos) & 1 if is_lon: mid = (lon_range[0] + lon_range[1]) / 2 if bit: lon_range[0] = mid else: lon_range[1] = mid else: mid = (lat_range[0] + lat_range[1]) / 2 if bit: lat_range[0] = mid else: lat_range[1] = mid is_lon = not is_lon lat = (lat_range[0] + lat_range[1]) / 2 lon = (lon_range[0] + lon_range[1]) / 2 return lat, lon # Example print(encode(28.6139, 77.2090, precision=6)) # "ttnfv2" (New Delhi) print(encode(19.0760, 72.8777, precision=6)) # "te7udw" (Mumbai) # These share no prefix — correct, they are ~1400 km apart`}} {{think "The alternating longitude/latitude bisection is what makes geographic proximity align with string prefix proximity — but only approximately."
