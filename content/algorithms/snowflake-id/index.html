"UUIDs are 128 bits and random. In a B-tree index, random inserts cause page splits on every write — terrible write performance. Snowflake IDs are time-sorted, so inserts always go to the rightmost leaf page — like an auto-increment but distributed. At scale this is a 5-10x write performance difference."
