{{phase 1 "What Is Object Storage?" "3 min"}} {{say `Object storage treats every file as an opaque blob with a key. Unlike a file system — no directories, no in-place updates, no partial writes — just three operations:
  • PUT — write an entire object (atomic, all-or-nothing)
  • GET — read an entire object by key
  • DELETE — remove an object
S3 stores objects in buckets. A bucket is a flat namespace — keys can contain slashes to look like paths, but S3 sees them as opaque strings.`}} {{diagram "fund-s3-overview"}} {{think "Object storage is the right answer whenever you are storing large blobs that are written once and read many times.
